# VEDA OS Product Brief

## North Star

VEDA OS helps a user hold the whole system of their life and work without flattening it into a task list. It should feel like a calm command center: present, opinionated, and alive to context.

## Primary Jobs

1. Orient: show what matters today and why.
2. Decide: turn loose thoughts, tensions, and opportunities into clear choices.
3. Execute: protect focus and translate priorities into next actions.
4. Remember: preserve the story behind decisions, projects, and commitments.
5. Reflect: turn completed work and lived experience into better future behavior.

## Initial Modules

- Today: daily focus, schedule pressure, rituals, next actions.
- Projects: active outcomes, status, blockers, commitments.
- Knowledge: notes, decisions, principles, source material.
- People: relationships, follow-ups, shared context.
- Signals: metrics, mood, energy, finances, health, and other operating data.
- AI Console: conversational planning and synthesis embedded into the system.

## First Milestone

Create a local-first browser prototype with a coherent information architecture, interactive module switching, seed data, and a visual language that feels like an operating system rather than a marketing site.

## Near-Term Product Questions

- Is VEDA personal-first, team-capable, or team-first?
- Should the core unit be projects, domains, commitments, or days?
- Which data should be local-only from day one?
- Where should AI act autonomously, and where should it only advise?
- What is the minimum daily ritual that makes the system valuable?
